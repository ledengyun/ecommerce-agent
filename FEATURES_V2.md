# 🎯 电商选品 Agent v2.0 - 新功能说明

## ✨ 新功能概览

### 1. 商品分类显示
- **未分析商品** - 显示：图片、标题、供货价、当前销售量
- **已分析商品** - 显示：图片、标题、供货价、建议售价、利润率、是否推荐

### 2. 智能分析逻辑
- **自动模式** - 不选择商品时，自动从未分析商品中选择（最多 10 个）
- **手动模式** - 可以选择特定商品进行分析
- **批量选择** - 支持全选未分析商品
- **智能提示** - 未分析商品不足时自动调整数量

### 3. 分析结果展示
- **只显示最近一次** - 面板上仅显示最后一次分析结果
- **实时统计** - 分析商品数、推荐数、平均利润率
- **进度显示** - 实时显示分析进度

## 🚀 使用流程

### Step 1: 查看商品

**未分析标签页：**
- 显示所有未分析的商品
- 可以看到供货价和当前销售量
- 可以勾选商品准备分析

**已分析标签页：**
- 显示所有已分析的商品
- 可以看到完整的利润信息
- 推荐商品有绿色标识

### Step 2: 选择商品（可选）

**方式 1：手动选择**
- 在未分析列表中点击商品
- 或勾选复选框
- 可以全选

**方式 2：不选择**
- 直接点击"开始分析"
- 系统自动选择 10 个未分析商品

### Step 3: 开始分析

点击"开始分析"按钮：
- 显示分析进度
- 实时更新状态
- 完成后显示结果

### Step 4: 查看结果

分析完成后：
- 右侧面板显示结果
- 自动刷新商品列表
- 已分析商品移动到"已分析"标签

## 💡 使用技巧

### 技巧 1：批量分析
1. 切换到"未分析"标签
2. 勾选"全选"
3. 点击"开始分析"
4. 一次性分析所有未分析商品

### 技巧 2：精准分析
1. 手动选择特定商品
2. 只分析感兴趣的商品
3. 节省分析时间

### 技巧 3：查看历史
- 结果框只显示最近一次
- 之前的结果可以在"已分析"标签查看
- 所有数据都保存在数据库

### 技巧 4：刷新数据
- 页面每 5 秒自动刷新统计
- 手动刷新可以按 F5
- 分析完成后自动刷新

## 🔧 技术细节

### API 端点

| 端点 | 功能 |
|------|------|
| `GET /api/stats` | 获取统计数据 |
| `GET /api/products/unanalyzed` | 获取未分析商品 |
| `GET /api/products/analyzed` | 获取已分析商品 |
| `POST /api/analyze` | 开始分析任务 |
| `GET /api/tasks/{id}` | 查询任务状态 |
| `GET /api/analysis/latest` | 获取最近分析结果 |

### 数据库字段

```sql
-- 商品表新增字段
analyzed BOOLEAN       -- 是否已分析
sales_volume VARCHAR   -- 销售量
last_analyzed_at TIMESTAMP -- 最后分析时间
```

### 分析逻辑

```python
# 1. 检查是否有选择的商品
if selected_products:
    analyze(selected_products)
else:
    # 2. 自动选择未分析商品（最多 10 个）
    products = get_unanalyzed(limit=10)
    analyze(products)

# 3. 计算利润
retail_price = supplier_price * 3.5
profit_margin = (retail - supplier - 5) / retail
recommend = profit_margin >= 0.3
```

## 🐛 常见问题

### Q1: 没有未分析商品怎么办？
**A:** 说明所有商品都已分析过了。可以：
- 导入新商品
- 或者重新分析已有商品（需要手动修改数据库）

### Q2: 分析失败怎么办？
**A:** 检查：
- 数据库连接是否正常
- 商品数据是否完整
- 查看日志文件 `logs/backend.log`

### Q3: 如何选择特定商品？
**A:** 
- 在未分析列表中点击商品
- 或使用复选框
- 选中的商品会高亮显示

### Q4: 分析结果在哪里？
**A:** 
- 右侧面板"最近分析结果"框
- 只显示最近一次的结果
- 历史结果在"已分析"标签查看
